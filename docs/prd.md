# 需求文档

## 1. 应用概述

### 1.1 应用名称
图片视频赏析Web应用

### 1.2 应用描述
一个图片视频内容赏析Web应用,支持普通用户上传图片视频内容,经管理员审核后发布到探索页供其他用户浏览。应用包含上传页面、个人中心页面、管理后台页面和探索页面四个核心模块,并包含勋章系统(用户端+管理端)、图集写真模块(前端展示页+管理配置页)、每日图集模块,新增独立存储/极速分级/自定义字段能力,基于现有 Vite + Supabase 架构实现。全站所有涉及文件上传的功能,统一使用 Cloudflare R2 存储服务,不使用 Supabase Storage。同时新增前端性能优化(骨架屏、图片懒加载、渐进式加载)、缓存策略优化(浏览器缓存、Service Worker、服务端缓存)、智能内容推荐与个性化系统、内容标签系统增强及相关内容推荐等能力。新增面向管理员的 RESTful API 模块,提供完整接口文档与开发辅助工具。新增全站图片/视频 Blob 功能,支持后台配置开关控制,并新增探索页缓存功能开关及缓存图片命中率统计。新增富文本编辑器模块,采用 wangEditor,支持文档分类功能、自定义字段插入与模板管理,模板支持自定义字段,自定义字段插入位置跟随当前光标位置。修复未绑定微信公众号用户的 openid 显示不正确问题(显示值为 unknown_user)。新增全局登录弹窗功能,用户在任意页面均可通过弹窗完成登录,无需跳转至独立登录页面。新增下载设置功能,支持壁纸与写真图集分开配置下载开关、积分消耗及最低下载权限,前端提供图片下载入口并纳入权限点体系,我的页面新增下载记录,积分记录过滤重复记录与后台同步记录。修复我的下载页失效图片/视频问题,优化点击预览体验,防止出现404错误。修复我的页面、消息通知页面、签到页面等页面进入初期左上角标题消失问题,改为显示返回按钮加标题的固定布局。修复探索页瀑布流布局中竖屏视频显示为横向的问题,确保竖屏视频在瀑布流中正确展示为竖屏布局。修复数据库函数 search_path 安全警告问题,为所有提示 function_search_path_mutable 的函数设置固定 search_path 参数。新增后台配置信息导入导出功能,支持导出所有库及图片素材(不包括用户数据),优化导入导出涉及 UUID 及权限约束等,避免数据库被约束导入失败,新增增量导入功能。新增写真管理的申请审核机制,支持开启/关闭申请开关,开启后所有用户上传图集需审核通过后根据图集内配置的用户管理等级进行权限划分,不受权限组限制;关闭申请开关则根据用户权限组进行权限划分。优化用户管理的角色划分,改为 pt、vip、svip、vvip、admin(管理员)五个等级。新增导航栏DIY布局样式功能,支持管理员自定义导航栏布局与样式,新增动画效果配置。新增上传页面解析导入功能,支持通过配置解析接口自动提取网址内容并导入。新增审核图片处理参数功能,全站所有涉及审核、查重、分级、分类、每日图集清理的页面(前端与后台)统一使用审核图片处理参数,优化每日图集清理完成后刷新又要重新清理的问题(清理结果未存储入库)。后台图集写真新增批量或全部调整权限等级功能,支持批量将图集调整为某一级别(pt/vip/svip/vvip),避免未分级而前端看不到的问题。新增综合代理模块,支持视频代理加速功能,新增智能路由选择与流量统计成本分析。每日图集模块新增多公众号独立密码推送功能,并新增用户输入密码后自动创建账户机制,后台默认创建用户并绑定微信公众号对应openid,默认用户名为GZH加5位随机数(避免重复,遇到重复重新生成),密码为123456,用户初次访问图集输入密码创建账户后弹窗提示修改密码,如果用户不修改则保持默认密码123456,完善与supabase登录用户的auth相结合,避免没有数据导致用户无法登录,后续用户使用对应openid和对应密码时,将记录加在该用户数据内。探索页管理员新增编辑功能,可对标题、图片地址、视频地址、标签、封面、分类进行编辑。新增刷新图单页功能,每次展示一张图片,支持手动刷新与自动播放旋转翻页,可设置在底部导航栏中间位置,点击中间凸起按钮可刷新该页面。新增多大图组件配置功能,支持管理员在后台配置多大图组件的默认图片内容,可指定第一张图、第n张图及最后一张图的固定内容,创建公众号草稿大多图时可对图片进行拖拽排序。新增全局彩蛋功能,支持管理员在后台投放彩蛋,用户在页面满足条件时可随机触发,彩蛋支持虚拟积分、实物或优惠券等奖励方式,中奖后给予页面展示,后台可查看彩蛋位置、当前状态及中奖记录。彩蛋生效页面改为可从路径中选择,支持参数的路径选择,路径选择界面新增确定按钮,包括所有页面路径。新增寻找特控⭐互动模块,将不定数量的特控⭐随机放置在某个页面内,用户可点击特控⭐进行收集,最先收集10个或满足相关条件的用户获得奖励,我的页面当开启特控⭐活动时会显示收集瓶,用来查看收集进度。特控⭐不是藏在屏幕上而是页面里,可能全显示可能一半显示,出现率可自行设置。新增特控⭐和彩蛋在页面的预览功能,方便管理员知道会出现在什么位置。优化勋章图标样式。修复开屏广告没有跳过按钮的问题。修复所有图片左上角参数日志位置错误问题,原本位于左上角的参数日志现在移位至中间位置,需恢复至左上角正确位置。

## 2. 功能模块

### 2.1 上传页面

#### 2.1.1 基础上传功能
- 普通用户可上传图片视频内容
- 所有上传操作统一使用 Cloudflare R2 存储服务
- 上传完成后,用户可在个人中心查看自己上传的内容
- 内容默认为待审核状态
- 上传时可为作品添加标签

#### 2.1.2 解析导入功能
- 上传页面新增「解析导入」功能入口,与常规上传功能并列展示
- 用户可输入外部网址(如图片/视频分享链接),系统自动解析并提取内容信息,完成导入
- 解析规则由管理员在后台配置,支持多接口配置与智能匹配
- 前端交互流程:
  1. 用户点击「解析导入」按钮,弹出解析导入弹窗
  2. 弹窗内展示输入框,占位符提示「请输入需要解析的网址」
  3. 用户输入网址后,点击「开始解析」按钮
  4. 系统根据后台配置的解析规则,匹配对应解析接口
  5. 解析过程中显示加载动画与提示「正在解析中,请稍候」
  6. 解析成功后,自动填充标题、标签、图片/视频URL至上传表单
  7. 用户可在自动填充后继续编辑或补充信息,点击「确认上传」完成导入
  8. 解析失败时,显示友好提示「解析失败,请检查网址是否正确或稍后重试」,允许用户重新输入
- 解析字段处理规则:
  - 标题字段:直接映射至上传表单的标题输入框
  - 标签字段:若为字符串,直接填充;若为数组,逐一提取并以逗号分隔填充至标签输入框
  - 图片/视频字段:若为单个URL字符串,直接填充;若为数组,逐一提取每个URL,支持批量导入
  - 其他字段:若后台配置了额外字段映射(如描述、作者等),同样自动填充至对应表单字段
- 移动端适配:解析导入弹窗宽度适配手机屏幕,输入框与按钮符合移动端操作习惯,解析结果预览列表支持滑动查看

### 2.2 个人中心页面
- 普通用户可查看自己上传的全部内容
- 显示每条内容的审核状态(待审核、审核通过、审核拒绝)
- 审核拒绝的内容,用户可重新修改后上传
- 收藏夹:展示用户收藏的图片内容
- 用户搜索功能:支持搜索其他用户
- 用户备注功能:对搜索到的用户添加备注信息
- 资料修改功能:用户可修改个人资料信息
- 邀请码生成:用户可生成无权限等级的邀请码
- 注册用户绑定:通过邀请码注册的用户与其自动绑定

### 2.3 我的页面优化
- 优化图片编辑图标和删除图标布局,减少空间占用
- 右上角增加消息通知功能
- 右上角齿轮图标点击打开二级菜单内容,用于设置用户偏好:
  - 排除类型:用户可选择自己不喜欢的内容类型,排除后这些类型的内容不再在探索页展示
  - 喜好类型:用户可选择自己喜欢看的内容类型,系统将优先推荐这些类型的内容
- 缩小图一之间的距离
- 新增「下载记录」入口,展示用户所有下载历史
- 页面顶部固定显示返回按钮与页面标题,标题为「我的」,不随页面滚动消失
- 当开启特控⭐活动时,我的页面显示收集瓶入口,用户点击可查看特控⭐收集进度,包括已收集数量、总目标数量、收集明细(每个特控⭐的收集时间与来源页面)
- 优化勋章图标样式,确保图标清晰美观,符合整体界面风格

### 2.4 极速整理模式
- 右上角按钮:待定区
- 下方主区域:整理至资源类型
- 资源类型内容:后台设置的类型内容,以两行模式展示
- 云空间定义:待整理区域
- 待定区:原设备(HUAWEI)区域
- 增加教程说明
- 筛选功能:只视频、只图片、全选
- UI参考图片:使用图片 image-2.png 作为极速整理模式UI参考(图二方式)

### 2.5 探索页面
- 采用瀑布流布局展示管理员审核通过的图片视频内容
- 页面自适应电脑和手机屏幕大小
- 每行展示3-4张图片
- 保留图片最长边进行展示
- 竖屏视频在瀑布流中正确展示为竖屏布局,不强制横向显示
- 探索分类:
  - 最新:按时间倒序展示最新上传的图片
  - 推荐:基于推荐体系为用户个性化推荐内容,接入智能推荐系统,替代原「热门」分类
- 顶部分类和标签改为右下角浮动图标,点击可弹窗选择分类或标签
- 右下角设置刷新按钮,点击可手动刷新随机列表
- 不喜欢按钮:用户可对不喜欢的内容进行标记,标记后该内容不再在探索页展示

#### 2.5.1 收藏与「不喜欢」交互优化
- 操作触发与视觉反馈:
  - 用户点击「收藏」按钮:图标立即切换为已收藏状态,同时触发微动效,视觉变化在点击后50毫秒内呈现,无需等待后台响应
  - 用户点击「不喜欢」按钮:按钮立即变灰且图标切换为禁用样式,配合简洁动效提示操作生效
  - 反馈完成后,按钮进入短暂禁用状态(约500毫秒),防止误操作
- 异常处理与用户提示:
  - 若后台异步操作失败:前端维持视觉反馈状态,避免界面回退造成困惑
  - 在页面非干扰区域(如底部角落)显示轻量级提示(例如:「网络异常,稍后自动重试」),2秒后自动消失
  - 系统在后台静默重试失败操作(最多3次),重试成功则无额外提示;若最终失败,记录日志供后续处理
- 前后端解耦保障:
  - 视觉反馈完全依赖前端本地状态,与网络请求分离
  - 使用乐观更新机制:假设操作成功,立即更新界面;仅当后台明确失败时,才考虑状态回退

#### 2.5.2 管理员前端快捷操作(探索页)
- 管理员登录后,在探索页每张作品卡片上显示管理员操作入口(悬浮或长按触发操作菜单)
- 操作菜单包含以下功能:升热度、降热度、取消推荐、取消显示、下架、删除、编辑
- 编辑功能:点击后弹出编辑弹窗,可对标题、图片地址、视频地址、标签、封面、分类进行编辑,编辑完成后点击「保存」按钮,修改内容实时生效并同步至数据库
- 操作需二次确认(删除、下架等不可逆操作)
- 操作结果实时反映在探索页列表

后台一级菜单「体系管理」拆分为以下五个独立一级菜单,各菜单下设二级子菜单:

**积分系统**
- 积分规则配置:配置各行为对应的积分获取/消耗规则
- 积分记录:查看全站用户积分流水记录
- 积分管理:管理员手动调整用户积分

#### 2.27.5 交互说明

**批量上传流程**
1. 管理员进入图集管理页,新建或选择已有图集
2. 点击「批量上传」,选择本地图片文件(支持多选)
3. 系统调用 Cloudflare R2 上传接口,将图片上传至 album_photos/[图集ID]/[图片名] 路径
4. 上传过程中显示进度条,支持断点续传
5. 上传完成后自动解析图片数量,生成缩略图,写入 album_photos 表

**极速分级流程(图二方式)**
1. 管理员进入图集详情页,进入极速分级模式
2. 交互参考 image-2.png 所示极速整理模式:顶部标签栏切换分级类型,下方网格展示图片,支持快捷键(如 1/2/3 对应普通/非限制/限制)或批量勾选后统一设置分级
3. 分级操作实时同步至 album_photos 表的 level 字段
4. 每次分级操作写入 album_photo_level_logs 日志表,记录操作人、时间、分级结果
5. 分级功能统一使用审核图片处理参数

**自动打包PDF下载流程**
1. 管理员在图集配置页开启「自动打包PDF下载」开关
2. 系统触发 Edge Function,将该图集下所有图片按排序打包生成PDF
3. PDF文件上传至 Cloudflare R2,路径格式为 album_pdfs/[图集ID]/[图集名称].pdf
4. 系统自动更新图集的 download_url 字段
5. 具备「图集写真 - 下载」权限点的用户可在前端下载该PDF

**自定义字段配置流程**
1. 超级管理员进入「自定义字段库」配置页,新增/编辑/删除字段
2. 编辑图集时,从字段库中选择需要的字段,填充对应值
3. 字段值以 JSON 格式存储在 photo_albums.custom_field_values 或 album_photos.custom_field_values 中
4. 前端展示页根据配置决定是否显示自定义字段

**分表数据与系统原有图片数据的隔离逻辑**
- 系统原有图片数据存储于原有图片表,Cloudflare R2 存储路径前缀为 photos/
- 写真图片数据存储于独立的 album_photos 表,Cloudflare R2 存储路径前缀为 album_photos/
- 两套数据在数据库层面完全隔离,不共享表结构
- 权限校验、RLS 策略分别独立配置,互不影响

**申请审核流程**
1. 用户上传图集时,若图集「开启申请开关」为 true,图集进入待审核状态
2. 管理员在管理后台审核图集,审核通过后,图集状态变更为已通过
3. 审核通过后,系统根据图集内配置的「用户管理等级」字段进行权限划分,不受用户权限组限制
4. 若图集「开启申请开关」为 false,则根据用户权限组进行权限划分,无需审核
5. 审核功能统一使用审核图片处理参数

**批量或全部调整权限等级流程**
1. 管理员进入后台图集写真管理页面
2. 勾选需要调整权限等级的图集(支持批量勾选或全选)
3. 点击「批量调整权限等级」按钮,弹出权限等级选择弹窗
4. 选择目标权限等级(pt/vip/svip/vvip),点击「确认调整」
5. 系统弹出二次确认提示,确认后执行批量调整
6. 调整完成后,前端展示页立即生效,避免未分级而前端看不到的问题
7. 调整操作记录日志,包含操作人、时间、调整前后权限等级

### 2.36 全站 Blob 功能

#### 2.36.1 功能概述
- 全站图片、视频资源支持 Blob 化处理
- 开启后,前端在加载图片/视频资源时,通过 Fetch 请求获取资源二进制数据,转换为 Blob URL 后再赋值给对应的 img/video 标签,防止资源原始 URL 直接暴露在 DOM 中
- 适用范围:探索页瀑布流图片、图片弹窗大图、图集写真图片、个人中心图片、管理后台图片预览等全站所有图片/视频展示场景

#### 2.36.2 后台配置
- 是否开启全站 Blob 功能开关(默认关闭),位于「站长工具」或「存储管理」下的「Blob与缓存配置」二级菜单
- 开关开启后,前端统一走 Blob 加载逻辑;关闭后,恢复原始 URL 直接加载
- 开关状态变更实时生效,无需重启服务

#### 2.36.3 前端实现要点
- 封装统一的 Blob 资源加载工具函数,供全站图片/视频组件调用
- 加载流程:前端读取后台 Blob 功能开关状态,开关开启时通过 Fetch 请求资源 URL,获取 Blob 数据,使用 URL.createObjectURL() 生成 Blob URL,赋值给 img/video 标签的 src 属性;开关关闭时直接使用原始资源 URL
- Blob URL 在组件卸载时调用 URL.revokeObjectURL() 释放内存,防止内存泄漏
- Blob 加载过程中显示骨架屏或占位图,加载完成后平滑切换
- 与图片懒加载策略兼容:仅对进入视口的图片触发 Blob 加载

- 移动端适配,确保在微信内访问体验流畅
- 新增多公众号独立密码推送功能:支持为不同微信公众号配置独立的每日图集访问密码,每个公众号的密码独立生成、独立推送,互不干扰
- 新增用户输入密码后自动创建账户机制:用户初次访问每日图集输入密码后,后台默认创建用户并绑定微信公众号对应openid,默认用户名为GZH加5位随机数(避免重复,遇到重复重新生成),密码为123456,用户初次访问图集输入密码创建账户后弹窗提示修改密码,如果用户不修改则保持默认密码123456,完善与supabase登录用户的auth相结合,避免没有数据导致用户无法登录,后续用户使用对应openid和对应密码时,将记录加在该用户数据内

#### 2.38.2 素材库管理
- 独立素材库,与图集写真模块(album_photos 表)完全分开,使用独立数据表 daily_gallery_materials

#### 2.38.3 每日图片配置与发布
- 发布数量配置:后台可配置每日发布图片数量(正整数,最小1张),配置项存储于站点配置表,修改后次日零点生效

#### 2.38.4 密码验证与访问控制
- 密码生成与管理:每日零点自动生成新的访问密码,密码为随机字符串(建议6位数字或字母数字混合),密码加密存储于数据库,防止明文泄露,密码有效期为1小时(从用户获取密码时刻起计算),过期后需重新向公众号获取,后台密码管理页面展示当日密码、生成时间、有效期,管理员可手动重置当日密码
- 多公众号独立密码推送:支持为不同微信公众号配置独立的每日图集访问密码,每个公众号的密码独立生成、独立推送,互不干扰,后台新增「多公众号密码配置」页面,管理员可为每个绑定的微信公众号单独配置密码生成规则(如密码长度、字符类型等),每日零点系统自动为每个公众号生成独立密码,并分别推送至对应公众号,用户向不同公众号发送关键词时,收到的密码为该公众号专属密码,密码验证时,系统根据用户来源公众号匹配对应密码进行校验,多公众号密码配置支持启用/禁用单个公众号的密码推送功能,禁用后该公众号不再推送每日图集密码
- 访问验证流程:用户访问每日图集页面,页面展示密码输入框,用户输入密码后,系统校验密码是否正确且在有效期内,验证通过展示当日图片列表,用户可浏览,验证失败显示友好提示(如「密码错误或已过期,请重新向公众号获取」),并提供重新获取指引,密码验证失败次数超过限制(如5次)后,触发短暂冷却期(如5分钟),防止暴力破解
- 未登录用户 openid 记录:用户通过微信公众号链接访问时,系统从链接参数或微信网页授权中获取用户 openid,访问日志记录 openid(含未登录用户),用于后续数据统计与用户注册后的历史记录同步,openid 获取失败时,访问日志中该字段留空,不使用任何占位值(如 unknown_user)
- 用户输入密码后自动创建账户机制:用户初次访问每日图集输入密码后,系统校验密码正确且在有效期内,后台自动创建用户账户并绑定微信公众号对应openid,默认用户名为GZH加5位随机数(避免重复,遇到重复重新生成),密码为123456,同时在supabase auth表中创建对应用户记录,确保用户可正常登录,用户初次访问图集输入密码创建账户后,系统弹窗提示用户修改密码,弹窗内容包含当前默认密码123456及修改密码入口,用户可选择立即修改密码或稍后修改,如果用户选择稍后修改或关闭弹窗,则保持默认密码123456,后续用户使用对应openid和对应密码登录时,系统将访问记录、下载记录等数据关联至该用户账户,确保数据完整性

#### 2.38.5 微信公众号交互流程
- 用户获取密码流程:用户向绑定的微信公众号发送指定关键词(如「今日图片」),公众号自动回复消息,内容包含当日访问密码(明文展示)、当日图集直接访问链接(点击可跳转到图片浏览页面)、密码有效期说明(如「密码1小时内有效」),用户点击链接或手动访问页面,输入密码后查看当日图片
- 多公众号独立密码推送:用户向不同公众号发送关键词时,收到的密码为该公众号专属密码,回复消息内容根据公众号配置动态生成,包含该公众号专属密码、访问链接及有效期说明
- 关键词配置:后台可配置触发密码回复的关键词列表(如「今日图片」「每日图集」等),关键词匹配支持精确匹配与模糊匹配两种模式,可在后台配置,回复消息模板可在后台自定义,支持短代码插入(如 {{daily.password}}、{{daily.link}}、{{daily.expire_time}})

- 图片浏览页:验证通过后展示当日图片列表,采用网格或瀑布流布局,支持点击图片查看大图(弹窗展示),页面顶部展示当日日期与图片总数,提供「历史记录」入口,可按日期查看历史发布图片(需重新验证对应日期密码,或管理员可配置历史图片是否需要密码),移动端适配,确保在微信内置浏览器中体验流畅,图片加载采用懒加载策略,优化加载速度

#### 2.38.8 数据库表设计
- daily_gallery_materials 表:每日图集素材库表
- daily_gallery_records 表:每日图集发布记录表
- daily_gallery_history 表:每日图集历史发布图片明细表(用于去重)
- daily_gallery_passwords 表:每日图集密码表,新增 wechat_account_id 字段,用于关联微信公众号,支持多公众号独立密码存储
- daily_gallery_access_logs 表:每日图集访问日志表
- daily_gallery_config 表:每日图集配置表
- wechat_account_password_config 表:微信公众号密码配置表,存储每个公众号的密码生成规则、推送状态等配置信息
- daily_gallery_auto_users 表:每日图集自动创建用户表,存储通过密码验证自动创建的用户信息,字段包括用户ID、openid、用户名、密码(加密存储)、创建时间、是否已修改密码等

#### 2.38.9 数据同步:openid 与注册用户关联
- 用户注册后,系统自动将其 openid 对应的历史访问记录(daily_gallery_access_logs 表中 openid 匹配的记录)关联至其账户(写入 user_id 字段)
- 关联逻辑通过 Supabase Edge Function 在用户完成微信公众号绑定时触发
- 关联完成后,用户可在个人中心查看其历史访问记录

#### 2.38.10 自动化流程(每日零点)
- 触发方式:通过 Supabase Edge Function 定时任务(Cron)在每日零点执行
- 执行步骤:为每个绑定的微信公众号生成独立的当日访问密码,写入 daily_gallery_passwords 表(关联 wechat_account_id);若发布模式为自动,从 daily_gallery_materials 表中随机抽取 daily_count 张 is_used = false 的素材;校验所选素材ID不在 daily_gallery_history 表中(双重去重保障);写入 daily_gallery_records 和 daily_gallery_history 表,更新素材 is_used 状态;若可用素材不足,记录告警日志并通知管理员

#### 2.38.11 用户输入密码后自动创建账户流程
- 用户访问每日图集页面,输入密码后点击「立即访问」
- 系统校验密码正确且在有效期内,同时检查该 openid 是否已关联用户账户
- 若 openid 未关联用户账户,系统自动创建用户:
  1. 生成默认用户名:GZH加5位随机数(如GZH12345),若用户名已存在则重新生成
  2. 设置默认密码:123456
  3. 在 daily_gallery_auto_users 表中写入用户记录,包含用户ID、openid、用户名、加密密码、创建时间、是否已修改密码(默认false)
  4. 在 supabase auth 表中创建对应用户记录,确保用户可正常登录
  5. 将该 openid 关联至新创建的用户账户
- 用户账户创建完成后,系统弹窗提示用户修改密码:
  - 弹窗标题:「账户已创建」
  - 弹窗内容:「您的账户已创建成功,默认密码为123456,建议您立即修改密码以保障账户安全」
  - 弹窗按钮:「立即修改」「稍后修改」
  - 用户点击「立即修改」:跳转至密码修改页面,用户输入新密码后保存,系统更新 daily_gallery_auto_users 表中 is_password_changed 字段为 true,同时更新 supabase auth 表中对应用户的密码
  - 用户点击「稍后修改」或关闭弹窗:保持默认密码123456,用户可在个人中心随时修改密码
- 后续用户使用对应 openid 和对应密码登录时,系统将访问记录、下载记录等数据关联至该用户账户,确保数据完整性

### 2.39 全局登录弹窗

- 登录弹窗触发后,当前页面内容保持不变,不发生路由跳转

- 弹窗居中展示,背景半透明遮罩覆盖全屏

#### 2.39.4 登录后状态同步
- 登录成功后,全局用户状态(登录态、用户信息、权限信息)立即更新
- 当前页面中依赖登录态的组件(如收藏按钮、上传入口、个人中心链接等)自动响应状态变化,无需页面刷新
- 若登录前用户触发了需要登录的操作(如点击收藏),登录成功后自动执行该操作

### 2.40 下载功能

#### 2.40.1 功能概述
- 前端新增图片下载功能,分为壁纸下载与写真图集下载两个独立模块,分开设置与计算
- 下载前向用户展示所需积分提示,用户确认后扣减积分并执行下载
- 下载功能纳入权限点体系,未具备对应权限点的用户不可触发下载
- 积分不足时提示用户当前积分余额及所需积分,引导用户获取积分

#### 2.40.2 前端下载入口
- 壁纸下载:下载入口位于图片弹窗/详情页,仅在后台壁纸下载开关开启时展示,用户点击下载按钮后,弹出积分消耗确认提示(展示本次下载所需积分及当前积分余额),用户确认后,系统校验权限点与积分余额,校验通过后扣减积分,触发图片下载,写入下载记录,下载文件格式保持原图格式,文件名使用图片原始名称或系统生成名称
- 写真图集下载:下载入口位于图集写真前端展示页,仅在后台写真图集下载开关开启时展示,下载逻辑与壁纸下载一致,积分消耗按写真图集独立配置计算,写真图集下载同时受「图集写真 - 下载」权限点控制

#### 2.40.3 权限点配置
- 权限点体系新增「壁纸下载」权限点,控制用户是否可下载壁纸图片
- 「图集写真 - 下载」权限点已存在,同时控制写真图集下载
- 权限点可在权限组配置及用户单独权限点配置中使用
- 下载功能同时受后台下载设置中「最低下载权限」约束,两者取更严格的限制

#### 2.40.4 积分扣减规则
- 壁纸下载:每次下载1张壁纸,按后台配置的积分数量扣减,与写真图集积分独立计算
- 写真图集下载:每次下载1张/1套写真图集,按后台配置的积分数量扣减
- 积分扣减在下载文件生成前执行,扣减成功后方可触发下载
- 积分扣减记录写入积分流水表,变更原因显示为「下载壁纸」或「下载写真图集」

#### 2.40.5 我的页面——下载记录
- 我的页面新增「下载记录」入口
- 下载记录列表展示字段:缩略图、文件名称、下载类型(壁纸/写真图集)、消耗积分、下载时间
- 支持按下载类型筛选(全部/壁纸/写真图集)
- 支持按时间倒序排列
- 点击记录可查看对应图片/图集详情
- 下载记录列表加载时,后端校验每条记录对应的资源URL是否有效,对失效资源在列表中标记失效状态(如缩略图置灰、显示「资源已失效」标签)
- 用户点击失效资源时,前端拦截预览操作,弹出友好提示:「该资源已失效,无法预览」,不触发404页面跳转
- 提供「删除失效记录」按钮,允许用户批量清理失效下载记录

#### 2.40.6 积分记录过滤规则
- 积分记录列表(用户端)不显示重复记录:同一操作在短时间内因系统重试等原因产生的重复积分流水,仅展示一条
- 积分记录列表(用户端)不显示后台同步记录:由管理员后台手动同步或批量操作产生的积分变更记录,不在用户端积分记录中展示
- 过滤规则仅影响用户端展示,后台积分管理页面仍可查看全量流水记录
- 过滤逻辑在前端查询时通过接口参数或后端过滤实现,确保用户端积分记录简洁、无冗余

#### 2.40.7 数据库表设计
