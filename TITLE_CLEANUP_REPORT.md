# 作品标题清洗与优化功能实施报告

## ✅ 功能概述

实现了作品标题的智能清洗和格式化功能，自动识别并过滤无效的标题字段（如长数字串、UUID、文件哈希等），同时限制标题显示长度，提升用户体验和界面美观度。

---

## 🎯 解决的问题

### 优化前
- ❌ 显示无意义的文件名：`IMG_20240315123456.jpg`
- ❌ 显示 UUID：`a1b2c3d4-e5f6-7890-abcd-ef1234567890`
- ❌ 显示文件哈希：`d41d8cd98f00b204e9800998ecf8427e`
- ❌ 显示过长的标题，影响布局
- ❌ 显示相机默认名称：`DSC00123`、`WechatIMG456`

### 优化后
- ✅ 自动识别并过滤无效标题
- ✅ 无效标题显示为"未命名"或"视频作品"
- ✅ 限制标题长度（默认 30 字符）
- ✅ 保留有意义的标题内容
- ✅ 超长标题自动截断并添加省略号

## 🔧 技术实现

### 1. 核心函数：`cleanTitle()`

**函数签名：**
```typescript
const cleanTitle = (name: string | null | undefined, maxLength = 30): string
```

**参数：**
- `name`：原始标题
- `maxLength`：最大长度（默认 30）

**返回值：**
- 清洗后的标题，如果无效则返回"未命名"

**实现逻辑：**
```typescript
const cleanTitle = (name: string | null | undefined, maxLength = 30): string => {
  if (!name) return '未命名';
  
  // 1. 移除文件后缀名
  let cleaned = name.replace(/\.[^/.]+$/, "");
  
  // 2. 定义无效模式
  const invalidPatterns = [
    /^[0-9]{8,}$/,                           // 纯数字（8位以上）
    /^[a-f0-9]{32}$/i,                       // MD5/SHA256 哈希
    /^[a-f0-9]{8}-[a-f0-9]{4}-[a-f0-9]{4}-[a-f0-9]{4}-[a-f0-9]{12}$/i, // UUID
    /^[a-zA-Z0-9_-]{20,}$/,                  // 长随机字符串
    /^IMG_[0-9]+$/i,                         // IMG_数字格式
    /^DSC[0-9]+$/i,                          // DSC数字格式
    /^[0-9]{4}-[0-9]{2}-[0-9]{2}_[0-9]{6}$/, // 日期时间格式
    /^Screenshot_[0-9_-]+$/i,                // Screenshot_时间戳
    /^WechatIMG[0-9]+$/i,                    // 微信图片默认名
    /^QQ图片[0-9]+$/,                        // QQ图片默认名
  ];
  
  // 3. 检查是否匹配无效模式
  for (const pattern of invalidPatterns) {
    if (pattern.test(cleaned)) {
      return '未命名';
    }
  }
  
  // 4. 去除前后空格
  cleaned = cleaned.trim();
  
  // 5. 如果清洗后为空，返回"未命名"
  if (!cleaned) return '未命名';
  
  // 6. 限制长度
  if (cleaned.length > maxLength) {
    return cleaned.substring(0, maxLength) + '...';
  }
  
  return cleaned;
};
```

### 2. 无效模式识别

| 模式 | 正则表达式 | 示例 | 说明 |
|------|-----------|------|------|
| 纯数字 | `/^[0-9]{8,}$/` | `20240315123456` | 8位以上纯数字 |
| MD5哈希 | `/^[a-f0-9]{32}$/i` | `d41d8cd98f00b204e9800998ecf8427e` | 32位十六进制 |
| UUID | `/^[a-f0-9]{8}-[a-f0-9]{4}-...$/i` | `a1b2c3d4-e5f6-7890-abcd-ef1234567890` | 标准UUID格式 |
| 长随机串 | `/^[a-zA-Z0-9_-]{20,}$/` | `xjk23lkj4lk2j3lk4j5lk6j7` | 20位以上随机字符 |
| IMG格式 | `/^IMG_[0-9]+$/i` | `IMG_1234` | 相机默认格式 |
| DSC格式 | `/^DSC[0-9]+$/i` | `DSC00123` | 相机默认格式 |
| 日期时间 | `/^[0-9]{4}-[0-9]{2}-[0-9]{2}_[0-9]{6}$/` | `2024-03-15_123456` | 时间戳格式 |
| Screenshot | `/^Screenshot_[0-9_-]+$/i` | `Screenshot_20240315_123456` | 截图默认名 |
| 微信图片 | `/^WechatIMG[0-9]+$/i` | `WechatIMG123` | 微信默认名 |
| QQ图片 | `/^QQ图片[0-9]+$/` | `QQ图片20240315` | QQ默认名 |

## 📊 清洗规则详解

### 规则 1：纯数字过滤
```typescript
/^[0-9]{8,}$/
```
- **匹配**：`20240315123456`、`1234567890`
- **不匹配**：`照片123`、`2024春游`
- **原因**：8位以上纯数字通常是时间戳或无意义ID

### 规则 2：哈希过滤
```typescript
/^[a-f0-9]{32}$/i
```
- **匹配**：`d41d8cd98f00b204e9800998ecf8427e`
- **不匹配**：`abc123`、`照片hash`
- **原因**：32位十六进制是 MD5/SHA256 哈希

### 规则 3：UUID 过滤
```typescript
/^[a-f0-9]{8}-[a-f0-9]{4}-[a-f0-9]{4}-[a-f0-9]{4}-[a-f0-9]{12}$/i
```
- **匹配**：`a1b2c3d4-e5f6-7890-abcd-ef1234567890`
- **不匹配**：`照片-2024`
- **原因**：标准 UUID 格式

### 规则 4：长随机串过滤
```typescript
/^[a-zA-Z0-9_-]{20,}$/
```
- **匹配**：`xjk23lkj4lk2j3lk4j5lk6j7`
- **不匹配**：`my_photo_2024`
- **原因**：20位以上纯字母数字下划线通常是随机生成的

### 规则 5：相机默认名过滤
```typescript
/^IMG_[0-9]+$/i
/^DSC[0-9]+$/i
```
- **匹配**：`IMG_1234`、`DSC00123`
- **不匹配**：`IMG_春游`、`我的DSC`
- **原因**：相机默认命名格式

### 规则 6：日期时间过滤
```typescript
/^[0-9]{4}-[0-9]{2}-[0-9]{2}_[0-9]{6}$/
```
- **匹配**：`2024-03-15_123456`
- **不匹配**：`2024春游`
- **原因**：纯日期时间格式无意义

### 规则 7：截图默认名过滤
```typescript
/^Screenshot_[0-9_-]+$/i
```
- **匹配**：`Screenshot_20240315_123456`
- **不匹配**：`Screenshot_我的桌面`
- **原因**：系统截图默认命名

### 规则 8：社交软件默认名过滤
```typescript
/^WechatIMG[0-9]+$/i
/^QQ图片[0-9]+$/
```
- **匹配**：`WechatIMG123`、`QQ图片20240315`
- **不匹配**：`微信图片`、`QQ照片集`
- **原因**：社交软件默认保存名称

## 📞 技术支持

### 常见问题

**Q1: 为什么我的标题被显示为"未命名"？**
- 检查标题是否匹配无效模式
- 如果是有意义的标题，请联系管理员添加例外规则

**Q2: 如何调整标题长度限制？**
- 修改 `cleanTitle()` 函数的 `maxLength` 参数
- 不同页面可以设置不同的长度

**Q3: 如何添加新的过滤规则？**
- 在 `invalidPatterns` 数组中添加新的正则表达式
- 确保正则表达式准确匹配目标模式

**Q4: 编辑标题时为什么还显示原始名称？**
- 编辑功能使用 `stripExtension()` 保留原始标题
- 这样用户可以看到并修改完整的原始名称

**Q5: 如何批量清洗历史数据？**
- 当前版本仅清洗显示，不修改数据库
- 未来可以添加批量清洗功能
