# Zonerama 库功能使用指南

## 快速开始

### 1. 访问 Zonerama 库

**路径**：管理后台 → 内容运营 → Zonerama 库

**权限要求**：管理员（admin）

### 2. 获取图片

**步骤**：

1. **输入相册 ID**
   ```
   在「相册 ID」输入框中输入 Zonerama 相册 ID
   例如：12345
   ```

2. **点击获取**
   ```
   点击「获取图片」按钮
   或按 Enter 键
   ```

3. **等待完成**
   ```
   显示加载状态：「获取中...」
   完成后显示：「获取成功！新增 50 张图片，跳过 5 张重复图片」
   ```

### 3. 管理图片

**选择图片**：
- 点击图片卡片选择/取消选择
- 点击「全选」按钮全选/取消全选
- 选中的图片显示蓝色边框

**批量操作**：
- **刷新**：重新加载图片列表
- **转到壁纸库**：将选中图片转移到壁纸库
- **转到写真库**：将选中图片转移到写真库
- **删除**：删除选中图片

### 4. 转移图片

**转移到壁纸库**：
1. 选择要转移的图片
2. 点击「转到壁纸库」按钮
3. 在对话框中点击「确认转移」
4. 等待完成：「转移成功！成功 10 张」

**转移到写真库**：
1. 选择要转移的图片
2. 点击「转到写真库」按钮
3. 在对话框中选择目标图集
4. 点击「确认转移」
5. 等待完成：「转移成功！成功 10 张」

## 注意事项

### 1. 配置要求

**必须配置**：
- 图集内图片列表接口（album_photo_api）
- 图片代理接口（photo_api）

**配置路径**：
- 后台管理系统 → 系统参数设置 → 存储管理 → 专享上传

### 2. 权限要求

**必需权限**：
- 管理员（admin）
- admin_library 权限

### 3. 操作限制

**获取图片**：
- 相册 ID 不能为空
- 相册必须存在
- 相册必须有图片

**转移图片**：
- 必须先选择图片
- 转移到写真库必须选择目标图集
- 已转移的图片不能再次转移

**删除图片**：
- 必须先选择图片
- 删除操作不可恢复
- 已转移的图片不受影响

### 4. 性能建议

**批量操作**：
- 建议每次转移不超过 100 张图片
- 建议每次删除不超过 100 张图片

**网络要求**：
- 稳定的网络连接
- 足够的带宽

### 5. 数据安全

**备份**：
- 转移前建议备份数据
- 删除前确认图片不再需要

**去重**：
- 系统自动去重
- 重复图片会被跳过

## 故障排除

### 问题 1：获取图片失败

**可能原因**：
1. 配置未设置
2. 相册 ID 错误
3. 接口不可访问
4. 网络问题

**解决方法**：
1. 检查配置是否正确
2. 检查相册 ID 是否正确
3. 检查接口是否可访问
4. 检查网络连接
5. 查看浏览器控制台日志

### 问题 2：转移图片失败

**可能原因**：
1. 未选择图片
2. 未选择目标图集（写真库）
3. 权限不足
4. 数据库错误

**解决方法**：
1. 确认已选择图片
2. 确认已选择目标图集（写真库）
3. 检查权限是否正确
4. 查看浏览器控制台日志
5. 重试操作

### 问题 3：图片不显示

**可能原因**：
1. 图片 URL 错误
2. 代理接口失效
3. 网络问题

**解决方法**：
1. 检查图片 URL 是否正确
2. 检查代理接口是否可访问
3. 检查网络连接
4. 刷新页面

### 问题 4：重复图片

**可能原因**：
1. 多次获取同一相册
2. 数据库去重失败

**解决方法**：
1. 系统会自动跳过重复图片
2. 查看成功提示中的「跳过 X 张重复图片」
3. 如果仍有重复，手动删除

## 最佳实践

### 1. 工作流程

**推荐流程**：
```
1. 配置接口
   ↓
2. 获取图片
   ↓
3. 预览图片
   ↓
4. 选择图片
   ↓
5. 转移图片
   ↓
6. 验证结果
```

### 2. 批量处理

**分批处理**：
- 每次处理 50-100 张图片
- 避免一次性处理过多图片
- 减少网络和数据库压力

### 3. 质量控制

**人工审核**：
- 转移前预览图片
- 确认图片质量
- 删除不需要的图片

### 4. 数据管理

**定期清理**：
- 定期删除已转移的图片
- 保持 Zonerama 库整洁
- 减少数据库存储
