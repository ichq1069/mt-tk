# 需求文档

## 1. 应用概述

### 1.1 应用名称
图片视频赏析Web应用

### 1.2 应用描述
一个图片视频内容赏析Web应用，支持普通用户上传图片视频内容，经管理员审核后发布到探索页供其他用户浏览。应用包含上传页面、个人中心页面、管理后台页面和探索页面四个核心模块，并包含勋章系统（用户端+管理端）、图集写真模块（前端展示页+管理配置页）、每日图集模块，新增独立存储/极速分级/自定义字段能力，基于现有 Vite + Supabase 架构实现。全站所有涉及文件上传的功能，统一使用 Cloudflare R2 存储服务，不使用 Supabase Storage。同时新增前端性能优化（骨架屏、图片懒加载、渐进式加载）、缓存策略优化（浏览器缓存、Service Worker、服务端缓存）、智能内容推荐与个性化系统、内容标签系统增强及相关内容推荐等能力。新增面向管理员的 RESTful API 模块，提供完整接口文档与开发辅助工具。新增全站图片/视频 Blob 功能，支持后台配置开关控制，并新增探索页缓存功能开关及缓存图片命中率统计。新增富文本编辑器模块，采用 wangEditor，支持文档分类功能、自定义字段插入与模板管理，模板支持自定义字段，自定义字段插入位置跟随当前光标位置。修复未绑定微信公众号用户的 openid 显示不正确问题（显示值为 unknown_user）。新增全局登录弹窗功能，用户在任意页面均可通过弹窗完成登录，无需跳转至独立登录页面。新增下载设置功能，支持壁纸与写真图集分开配置下载开关、积分消耗及最低下载权限，前端提供图片下载入口并纳入权限点体系，我的页面新增下载记录，积分记录过滤重复记录与后台同步记录。

## 2. 版本管理规则
- **云函数更新**：涉及云函数修改，必须更新云函数版本号。
- **后台更新**：涉及后台管理功能修改，必须更新后台版本号。
- **前台更新**：涉及前台功能修改，必须在首页左上角站名后显示版本号。
- **记录要求**：所有版本更新规则需记录在需求文档中。
- **更新日志**：每次更新需在需求文档末尾记录更新日志。

## 3. 更新日志

### [v2.0.03] - 2026-03-29
#### 修复与优化
- **数据库函数**: 修复 `record_cache_hit` 函数强制删除与重新创建，并配置正确的权限（GRANT EXECUTE TO anon, authenticated）。
- **后台一致性**: 优化后台配置保存逻辑，保存后强制刷新页面/配置状态，确保数据即时生效。
- **数据存储**: 将配置保存逻辑从 `insert` 统一优化为 `upsert`，避免重复数据并支持更新。
- **版本控制**: 按照新规则初始化版本管理体系，在前台首页显示版本号，并建立 `需求文档.md` 及更新日志。


### [v2.0.04] - 2026-03-29
#### 新增功能
- **微信消息对话记录**: 微信消息列表新增“对话记录”功能，支持查看特定用户与公众号的历史交互流水（气泡对话模式）。
- **消息搜索**: 微信消息列表新增 OpenID 搜索功能，方便精准定位特定用户的交互记录。
- **对话追踪**: 优化对话记录展示，包括用户发送的消息、触发的事件以及系统的自动回复。

### [v2.0.05] - 2026-03-29
#### 微信公众号草稿系统增强
- **已入稿库收集优化**: 修复“已入稿库”未包含“已采用”图片的问题，现在已采用的草稿图片也会同步显示在已入稿库中。
- **正文编辑器修复**: 修复从草稿媒体库应用模板时图片显示为空的问题。通过添加 `referrerPolicy="no-referrer"` 确保微信图片在编辑器内正常渲染，并优化了模板解析逻辑。
- **单次使用原则**: 完善图片状态管理逻辑。每张图片在微信草稿中仅限使用一次，一旦进入“已入稿”或“已采用”状态，即刻从“草稿媒体库”的可选列表中移除，避免重复使用。
- **移出名单功能**: 媒体库新增“移出名单”（排除）功能。用户可手动将不合适的图片标记为“已排除”，这些图片将不再出现在媒体库采集池中。
- **操作指南与说明**: 在管理界面新增“操作指南”菜单，详细解释媒体库、入稿库、排除名单等模块的逻辑流转，防止操作冲突。
- **批量创建逻辑修复**: 优化批量创建草稿时的正文存储格式，确保在数据库中保存为标准的区块 JSON，使批量创建的草稿能被编辑器正确二次编辑。
