## 技术说明

### 本次修复的关键改进
1. **Dialog 强制重新挂载**
   - 使用 `key={editingAd?.id || 'new'}` 
   - 每次编辑不同广告时完全卸载旧 Dialog

2. **打开前完全清理**
   - 先关闭 Dialog 并清空状态
   - 延迟 100ms 后再打开
   - 确保 DOM 完全清理

3. **保存后延迟刷新**
   - 先关闭 Dialog
   - 等待 300ms（动画时间）
   - 再刷新列表

### 为什么需要延迟
- Dialog 关闭动画需要时间（约 200-300ms）
- React Portal 组件需要时间完全卸载
- 浏览器需要时间完成渲染周期
- 延迟确保 DOM 操作在稳定状态下进行

## 常见问题

### Q: 为什么点击编辑后有轻微延迟？
A: 这是正常的，100ms 的延迟用于确保 DOM 完全清理，用户几乎无感知。

### Q: 保存后列表刷新有延迟？
A: 这是正常的，300ms 的延迟等待 Dialog 关闭动画完成，体验更自然。
